üéØ Problem to be Solved

Background

Wine quality assessment is traditionally performed by human experts through sensory evaluation, which presents several challenges:

Traditional Challenges

  • Subjective: Varies between different tasters
  • Time-consuming: Requires trained professionals
  • Expensive: Limited scalability for large production
  • Inconsistent: Human fatigue affects results

Our Solution

  • Objective: Based on measurable chemical properties
  • Fast: Instant predictions
  • Scalable: Process thousands of samples
  • Consistent: Same input = same output
3

Why This Problem Matters

  • Applicable to both red and white wine varieties

ü§ñ Algorithms Implemented

üìà Linear Regression

Simple baseline model that finds linear relationships between features and quality

  • Fast training
  • Interpretable coefficients
  • Assumes linear relationships

üå≤ Random Forest

Ensemble of 100 decision trees with bootstrap aggregating

  • Handles non-linear relationships
  • Feature importance ranking
  • Robust to overfitting

üéØ K-Nearest Neighbors

Instance-based learning with K=10 neighbors

  • No training phase
  • Non-parametric method
  • Sensitive to feature scaling

üå≥ Decision Tree

Single tree model for interpretable decision rules

  • Easy to interpret
  • Visual representation
  • Prone to overfitting

üìè Performance Metrics Explained

MSE

Mean Squared Error

Average of squared differences between predicted and actual values

Lower is better ‚Üì

Penalizes large errors more heavily

MAE

Mean Absolute Error

Average absolute difference between predicted and actual values

Lower is better ‚Üì

Easy to interpret in original units

R¬≤

Coefficient of Determination

Proportion of variance in quality explained by the model

Higher is better ‚Üë

Range: 0 to 1 (1 = perfect)

Why Random Forest Won

  • Lowest MSE (0.3599): Fewer large prediction errors
  • Lowest MAE (0.4357): Best average prediction accuracy
  • Highest R¬≤ (0.5108): Explains most variance in quality scores
