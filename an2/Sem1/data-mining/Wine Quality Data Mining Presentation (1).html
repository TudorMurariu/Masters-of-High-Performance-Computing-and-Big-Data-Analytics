Wine Quality Prediction

A Data Mining Approach Using Multiple Algorithms

Data Mining Course Project

1

Problem to be Solved

The Challenge

Wine quality assessment traditionally relies on human sensory evaluation by expert tasters, which is:

  • Subjective and inconsistent
  • Time-consuming and expensive
  • Limited by human availability

Our Goal

Predict wine quality scores (3-9) based on physicochemical properties using data mining algorithms.

Why This Matters

  • Automate quality control in wineries
  • Identify key factors affecting quality
  • Support winemakers' decisions
  • Reduce costs and time

Task Type

Regression Problem - Predicting a continuous quality score from chemical features.

Implemented Algorithms

1. Linear Regression

Simplest approach - finds linear relationship between features and quality.

Assumes: Linear relationship, feature independence

2. Random Forest

Ensemble of 100 decision trees. Each tree votes, final prediction is average.

Parameters: n_estimators=100

3. K-Nearest Neighbors (KNN)

Predicts based on average quality of K most similar wines.

Optimal K=10 (found via validation)

Conclusions

Main Findings

  • Random Forest achieved best performance (R¬≤ = 0.51, MAE = 0.44)
  • Alcohol content is the strongest predictor of wine quality
  • Ensemble methods outperform simple models for this task
  • Wine type (red/white) is NOT a significant factor for quality prediction

Algorithm Ranking

  1. Random Forest (Best)
  2. KNN
  3. Linear Regression
  4. Decision Tree

Practical Implications

  • Unified model works for both wine types
  • Focus on alcohol & acidity in quality control
  • Average prediction error: ~0.5 quality points
